# AirBalance Pose Analyzer

YOLO11n-pose 모델 기반의 실시간 하체 포즈 분석 및 맞춤형 안전 임계각 산출 프로그램입니다. 사용자의 다리 길이를 학습하여 개인화된 안전 임계 범위(보폭 각도)를 설정하고 실시간 자세 모니터링을 가능하게 합니다.


## 1. 파일별 역할 정의

### main.py (메인 컨트롤러)
주요 역할: 비디오 캡처, AI 분석, 렌더링을 조율하는 메인 루프 실행
핵심 연동 컴포넌트: run_pipeline() 함수

### constants.py (공통 상수 설정)
주요 역할: 프로그램 전반에 필요한 환경 설정값 및 포즈 인덱스 보관
핵심 연동 컴포넌트: InitState 클래스 및 전역 상수

### init_manager.py (사용자 상태 관리)
주요 역할: 사용자 다리 길이 측정, 개인화 안전 각도 계산 및 리셋 감지
핵심 연동 컴포넌트: InitializationManager 클래스

### yolo_processor.py (인공지능 이미지 분석)
주요 역할: YOLO11n-pose 모델 기반 사용자 추적 및 관절 좌표 추출
핵심 연동 컴포넌트: YOLOProcessor 클래스

### video_source.py (입력 영상 소스 제어)
주요 역할: 웹캠, 로컬 파일, 유튜브 주소 등 비디오 스트림 연동
핵심 연동 컴포넌트: VideoSourceManager 클래스

### visualizer.py (화면 피드백 시각화)
주요 역할: 화면에 스켈레톤, 게이지 바, 임계값 텍스트 그래픽 렌더링
핵심 연동 컴포넌트: Visualizer 클래스

## 2. 프로그램 동작 흐름

전체 파이프라인은 아래와 같이 5단계의 상태 변화를 거쳐 실행됩니다.

[1단계] 시스템 로드 및 준비
- 입력 영상 소스를 연결하고 YOLO 포즈 추정 가중치 파일(yolo11n-pose.pt)을 메모리에 불러옵니다.
- 초기화 상태를 NONE(대기)으로 설정하고 실시간 프레임 분석을 시작합니다.

[2단계] 사용자 대기 (NONE)
- 카메라 화면에 사람이 들어오기를 기다립니다. (화면에 "Waiting for person..." 표시)
- 사람이 감지되면 감지된 사용자의 고유 트래킹 ID를 기록하고 자동으로 IN_PROGRESS 상태로 넘어갑니다.

[3단계] 사용자 신체 측정 (IN_PROGRESS)
- 7초 동안 매 프레임마다 사용자의 엉덩이 관절부터 발목까지의 거리를 계산하여 다리 길이를 수집합니다.
- 화면 하단에 0%부터 100%까지 게이지 바를 그리며 진행 상황을 보여줍니다.
- 측정 도중 다른 사람으로 바뀌면 데이터 수집을 중단하고 처음부터 다시 측정을 시작합니다.

[4단계] 계측 완료 및 분석 모드 진입 (DONE)
- 7초간 수집한 다리 길이의 평균값을 내고 이를 바탕으로 개인화된 안전 임계각을 계산합니다.
- 계산이 끝나면 화면에 "Ready | Threshold: 각도"를 띄우고 실시간 자세 및 위험도 측정 단계로 들어갑니다.

[5단계] 이탈 및 사용자 교체 감지
- 화면에 3초 이상 사람이 감지되지 않으면 기구를 비운 것으로 보고 NONE 상태로 리셋합니다.
- 기구를 비우지 않았더라도 감지된 사람의 트래킹 ID가 바뀌거나 실시간 다리 길이가 기존 측정값 대비 15% 이상 급격히 달라지면, 1.5초 대기 후 자동으로 NONE 상태로 초기화하여 새 사용자의 신체를 다시 계측합니다.


## 3. 핵심 상수 및 변수 정리

### constants.py (시스템 임계값 설정)

- INIT_DURATION (실수형): 신체 데이터 계측에 소요되는 시간 (기본값 7.0초)
- ABSENCE_THRESHOLD (실수형): 리셋 처리를 위한 사용자 미감지 허용 시간 (기본값 3.0초)
- ALPHA (실수형): 다리 길이 대비 안전 보폭 허용 비율 (기본값 0.8)
- CONF_THRESHOLD (실수형): YOLO 관절 검출 신뢰도 최소 컷오프값 (기본값 0.5)
- LEG_LENGTH_TOLERANCE (실수형): 사용자 변경 판정을 위한 다리 길이 오차 허용 비율 (기본값 0.15)
- MISMATCH_THRESHOLD (실수형): 사용자 불일치 감지 시 최종 리셋까지 대기하는 유예 시간 (기본값 1.5초)

### InitializationManager (사용자 상태 관리)

- state: 현재 계측 진행 상태 (NONE, IN_PROGRESS, DONE 중 하나)
- leg_length_px: 7초간 수집하여 계산된 사용자의 평균 다리 길이 픽셀 값
- threshold_angle: 기하학 수식으로 최종 계산된 사용자의 안전 경보 임계각
- current_track_id: 현재 계측하여 분석하고 있는 사용자의 고유 트래킹 ID (미할당 시 None)
- landmark_buffer: 7초 동안 매 프레임의 다리 길이 데이터를 임시 저장하는 리스트
- init_start_time: 신체 계측이 시작된 시점의 시스템 타임스탬프
- absence_start_time: 사용자가 화면에서 사라진 시점의 시스템 타임스탬프
- mismatch_start_time: 다른 사용자로 의심되는 불일치가 발생한 시점의 시스템 타임스탬프

### YOLOProcessor (인공지능 분석 결과 딕셔너리)

- keypoints: 감지된 사람의 17개 관절 좌표와 검출 신뢰도 정보가 담긴 튜플 리스트
- bbox: 감지된 사람 영역의 바운딩 박스 화면 좌표 튜플 (x1, y1, x2, y2)
- bbox_area: 바운딩 박스의 전체 면적 픽셀 값
- track_id: 프레임 간 동일 인물을 추적하기 위해 모델이 발급한 고유 정수 ID (트래커 미할당 시 None)
